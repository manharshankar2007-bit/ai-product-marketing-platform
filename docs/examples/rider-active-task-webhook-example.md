Rider Active Task Webhook

Bridge the gap between dispatch and execution with real-time, event-driven visibility into your riders' active tasks.

You can now subscribe to webhooks that provide structured updates on a rider's ongoing task sequence—enabling your systems to stay in sync without relying on live tracking UI.

**What's New**

- **Real-Time Task Visibility**
  Get structured updates on all active tasks assigned to your riders, including deliveries and admin tasks.

- **Task Sequence Clarity**
  Understand not just what the rider is doing, but also their current stage and upcoming tasks in the execution journey.

- **SLA Risk Monitoring**
  Identify potential delays early with continuous updates on active task execution.

Go to *Settings* → *Communications* → *Rider*

1. Select **Rider Active Task** trigger
2. Enable the webhook configuration
3. Add:
   - Webhook URL
   - Auth Token
4. Test the webhook
5. Save the configuration once the test is successful

**Why This Matters To You**

- **System-Level Integration**
  Easily integrate rider task data into your internal systems using webhook-based updates.

- **Clean & Relevant Data**
  Webhook payloads include only active/ongoing tasks. Completed or terminal tasks are excluded to ensure signal over noise.

**What changes?**

1. Nothing changes in day to day operations
2. New Webhook (completely optional) can be subscribed
3. Updated SLAs include the actual time spent in each order (including pre-decided waiting times for both pickup and drop-off).
